post /jobs/{jobId}/action
Perfrom an action on job given *jobId* and *action* (jobsJobIdActionPost)
Perfrom an action on job given job id. A action can only be either CANCEL, RESTART, PAUSE or RESUME.

Path parameters

jobId (required)
Path Parameter — Job id.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/x-www-form-urlencoded

Form parameters

action (required)
Form Parameter — The action applied on the job.

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

200

Action applied on job.

404

If there is no job with given jobId

500

Server Internal Error.

503

If there is an issue connecting to DynamoDB.

post /jobs
Create a new job (jobsPost)
Create a new job given *userEmail* and original *documentURL*. The original document must be shared with [sheetapi@bingdatacleaner.iam.gserviceaccount.com](mailto:sheetapi@bingdatacleaner.iam.gserviceaccount.com) If a new job is created, return the new job json.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/x-www-form-urlencoded

Form parameters

userEmail (required)
Form Parameter — User email for job status notification.
documentUrl (required)
Form Parameter — The URL to shared original document.

Return type

Object

Example data

Content-Type: application/json
"{}"

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • application/json

Responses

201

Job created and a new job object is returned. Object

412

If user did not give permission for the service to access the source document.

500

Server Internal Error.

503

If there is an issue connecting to DynamoDB.
